## 1. 背景与问题定义

AyaFeed 当前首页已经具备明确的报纸品牌风格与时间轴内容组织能力，但它仍然更像“带头条卡的时间轴页”，而不是一个成熟的 landing page。当前版本的主要问题是：

1. 首屏优先展示“单个活动”，但没有先讲清楚 AyaFeed 是什么、对谁有价值、下一步应该做什么。
2. 首页的主要浏览路径偏单一，用户需要先进入时间轴，才能理解产品的核心能力。
3. 工具入口不够前置，搜索、筛选、地区、类型等检索心智没有在首屏清晰建立。
4. 供给侧入口较弱，社团与主办方很难在首页快速理解 AyaFeed 能提供什么能力。
5. Header、Footer、Logo 已有基础，但仍偏“页面组件”，尚未形成完整的首页品牌系统。

本次重构的目标，是把 AyaFeed 首页升级为：

- 一个有明确品牌气质的编辑型首页
- 一个能快速承接检索任务的工具型入口
- 一个能让观众、社团、主办方都看懂路径的参与型首页

## 2. 产品定位

### 2.1 首页定位

AyaFeed 首页不是售票平台首页，也不是纯内容杂志首页。它的定位应为：

**“东方活动情报首页 + 档期索引入口 + 参与路径入口”**

### 2.2 推荐路线

本次重构采用以下混合路线：

- 外层结构借鉴 `The New York Times` 的编辑式首页秩序
- 工具入口借鉴 `Meetup / AllEvents / PassMarket` 的搜索与筛选前置策略
- 用户路径借鉴 `hamp` 的任务导向入口设计
- 局部行为借鉴 `东方市场` 的明确动作承接

### 2.3 一句话价值主张

建议首页主张统一为：

**最快找到近期东方同人展、演出与社团情报**

补充说明文案建议：

**按地区、时间与类型快速检索，也可提交活动与社团信息，进入 AyaFeed 收录体系。**

## 3. 目标用户

### 3.1 核心用户

- 观众 / 参与者：查近期活动、看重点档期、寻找社团与演出信息
- 社团：希望获得曝光、展示资料、沉淀页面
- 主办方：希望提交活动、补充信息、建立可信入口

### 3.2 次级用户

- 内容编辑 / 维护者：需要通过首页传达“内容在持续更新”
- 第一次访问 AyaFeed 的新用户：需要在 5 秒内理解 AyaFeed 的价值

## 4. 重构目标

### 4.1 业务目标

- 提升首页首次访问用户的理解率
- 提升首页到 `/events`、`/lives`、`/circles` 的点击转化
- 提升“提交活动情报 / 社团入驻”类供给侧入口的可见度
- 提升首页的品牌识别度与可信度

### 4.2 体验目标

- 首屏 5 秒内回答 3 个问题：
  - 这是什么？
  - 我能做什么？
  - 下一步点哪里？
- 让桌面端更像“头版首页”，移动端更像“任务优先入口”
- 让首页从“长内容滚动页”变成“可浏览、可检索、可参与”的结构化首页

### 4.3 成功指标

建议埋点并跟踪：

- Hero 主 CTA 点击率
- 首页搜索框交互率
- 首页筛选器使用率
- 首页到 `/events` 的点击率
- 首页到 `/feedback` 的点击率
- 首页到 `/circles` 的点击率
- 首页首屏停留时长
- 首页滚动深度

## 5. 非目标

本次重构不包含以下内容：

- 不把首页升级成售票平台
- 不在首页直接引入复杂账户系统流程
- 不把首页做成完整活动列表页
- 不引入第二套视觉主题
- 不在本次阶段重做底层数据结构

## 6. 信息架构

推荐首页信息架构如下：

1. Utility Bar
2. Masthead
3. Primary Navigation + Search
4. Hero（价值主张 + 主头条）
5. Secondary Picks
6. Command Bar（搜索 / 地区 / 月份 / 类型）
7. 今日 / 本周焦点
8. 本月重点档期
9. 最近更新 / 新收录
10. 精简版时间轴
11. 角色入口（观众 / 社团 / 主办）
12. Footer

## 7. 核心用户路径

### 7.1 观众路径

首页进入后，优先看到：

- AyaFeed 是什么
- 近期重点活动
- 搜索与筛选能力
- 进入完整活动列表或具体详情

推荐路径：

`首页 -> 浏览近期活动 -> 活动详情 / 演出详情 / 社团详情`

### 7.2 社团路径

首页进入后，优先看到：

- AyaFeed 不只是看活动，也支持社团曝光
- 存在清晰的社团入口
- 可以进入社团入驻说明或社团列表

推荐路径：

`首页 -> 我是社团 -> 社团入驻说明 / 社团列表`

### 7.3 主办路径

首页进入后，优先看到：

- AyaFeed 支持活动收录与提交
- “提交活动情报”是高优先级动作
- 有收录说明与反馈入口保障可信度

推荐路径：

`首页 -> 提交活动情报 -> 收录说明 / 反馈`

## 8. 模块级设计要求

### 8.1 Utility Bar

#### 目标

建立“内容持续更新”的感知，并提供轻量全局操作。

#### 内容

- 最近更新日期
- 已收录数量
- 本周新增数量
- 地区切换
- 语言切换
- 提交活动情报

#### 设计要求

- 高度控制在 32–36px
- 低视觉权重，但文本对比度必须足够
- `提交活动情报` 是唯一高对比主动作
- 桌面端完整展示，移动端可压缩成 1 行

#### 不应出现

- 过多说明文案
- 过多装饰性元素
- 低对比浅灰文本

### 8.2 Masthead

#### 目标

建立首页品牌锚点，让 AyaFeed 更像一个可信的“情报首页”。

#### 内容

- Symbol + Wordmark：`文文。速报`
- 副标题：`幻想乡活动情报总览`

#### 设计要求

- Wordmark 优先级高于 Symbol
- 默认静态展示，不做持续动画
- 保留 newspaper 调性，但简化多余装饰

### 8.3 Primary Navigation + Search

#### 目标

建立全站一级入口，并前置搜索能力。

#### 导航项

- 展会
- 演出
- 社团
- 归档
- 收录说明
- 关于

#### 搜索框

- 占位文案：`搜索活动 / 社团 / 地点`
- 桌面端常驻显示
- 移动端可使用图标触发搜索层

#### 设计要求

- 搜索必须在首屏可见
- `收录说明` 必须是一级可达
- 避免让地区与语言操作压过主导航

### 8.4 Hero（价值主张 + 主头条）

#### 目标

在首页首屏同时完成“产品理解”和“内容证明”。

#### 结构

左侧：

- Eyebrow：`本周头条 / Editor's Pick`
- H1：`最快找到近期东方同人展、演出与社团情报`
- 说明文案
- CTA 组
- Mini trust line

右侧：

- 主头条活动海报
- 标题
- 日期
- 地点
- 类型标签
- 查看详情

#### CTA

- 主 CTA：`浏览近期活动`
- 次 CTA：`按地区检索`
- 辅助 CTA：`提交活动情报`

#### 设计要求

- 左右布局优先级清晰
- 不允许海报压倒产品文案
- 文案应先解释 AyaFeed 是什么，再交给内容证明

### 8.5 Secondary Picks

#### 目标

承接 Hero，提供第二层点击路径。

#### 推荐内容

- 次重点活动 A
- 次重点活动 B
- 最近更新 / 新收录摘要

#### 设计要求

- 视觉权重低于 Hero
- 不要三张都用强视觉大图
- 第三张建议偏文本或系统更新感

### 8.6 Command Bar

#### 目标

建立 AyaFeed 的“可搜索、可筛选、可检索”认知。

#### 组成

- 搜索输入框
- 地区筛选
- 月份筛选
- 类型筛选
- 查看全部

#### 设计要求

- 桌面端一行完整展示
- 移动端拆成搜索入口 + 筛选 chips
- 是首页的工具承接层，不可隐藏过深

### 8.7 今日 / 本周焦点

#### 目标

用更短路径呈现当前最值得浏览的活动内容。

#### 逻辑

- 若存在今日活动，则标题为：`今日进行中`
- 若不存在今日活动，则标题自动切换为：`本周焦点`

#### 内容

- 2 张焦点卡
- 每张卡包含标题、日期、地点、类型、简短摘要、详情入口

#### 设计要求

- 不出现“今日情报”但列表为空的违和状态
- 摘要不超过 2 行
- CTA 明确且统一

### 8.8 Quick Rail

#### 目标

为首页正文提供辅助导航与可信信息，不与正文争夺主舞台。

#### 仅保留 3 类内容

- 本周统计
- 下一场重点活动
- 收录说明 / 最新更新

#### 设计要求

- 不再展示热门标签
- 不再展示伪广告位
- 右栏必须高度克制、信息明确

### 8.9 本月重点档期

#### 目标

建立 AyaFeed 的“编辑选择感”，让首页更像策展型头版。

#### 内容

- 4 张精选活动卡
- 字段包括：海报、标题、日期、地点、标签、详情按钮

#### 设计要求

- 控制在 4 张左右
- 是“精选区”，不是完整列表
- 适合桌面四列、移动横滑

### 8.10 最近更新 / 新收录

#### 目标

建立 AyaFeed 的内容活性与维护感。

#### 推荐内容

- 新收录活动
- 新收录演出
- 新入驻社团
- 信息变更

#### 设计要求

- 优先采用轻量文本卡，不依赖大图
- 每项只保留一行摘要 + 一个入口
- 明确告诉用户“这个站在持续更新”

### 8.11 精简版时间轴

#### 目标

保留 AyaFeed 的差异化内容资产，但不让时间轴继续垄断首页。

#### 内容

- 月份跳转
- 近期少量条目预览
- 进入完整时间轴 / 历史归档

#### 设计要求

- 首页只展示预览，不展示完整长流
- 每月展示少量卡片即可
- 移动端海报比例缩小，降低滚动成本

### 8.12 角色入口（Role Entry）

#### 目标

把 AyaFeed 从“信息浏览站”升级成“有参与路径的首页”。

#### 角色卡

- 我是观众：查活动、演出、社团
- 我是社团：申请入驻、展示资料
- 我是主办：提交活动、申请收录

#### CTA

- `开始检索`
- `查看社团入驻`
- `提交活动情报`

#### 设计要求

- 每张卡一句话、一个按钮
- 不做复杂说明
- 是产品入口，不是营销广告

### 8.13 Footer

#### 目标

建立品牌、信任、说明与全局补充导航。

#### 四列结构建议

- 浏览：展会 / 演出 / 社团 / 归档
- 参与：提交活动 / 社团入驻 / 问题反馈
- 说明与信任：收录说明 / 数据来源 / 隐私政策 / 关于 AyaFeed
- 联系：联系渠道 / 社交链接

#### 必须包含

- 最近更新时间
- 数据覆盖说明
- 品牌说明

#### 不建议保留

- 指回首页的“关于我们”伪入口
- 低对比装饰性文案占主导

## 9. Header / Footer / Logo 评估结论

### 9.1 Header 结论

建议重构。

原因：

- 当前 header 的品牌层、工具层、导航层边界不够清晰
- 移动端地区切换占据首屏过多空间
- 搜索能力不够前置
- 可访问名称存在优化空间

### 9.2 Footer 结论

建议重构。

原因：

- 当前 footer 更像通用收尾区，而不是可信信息层
- 缺少对 AyaFeed 数据来源、收录逻辑、更新时间的补充说明
- 导航价值不足，信任价值不足

### 9.3 Logo 结论

不建议推翻重做，但建议扩展成完整品牌系统。

建议拆成三层：

1. Symbol：当前 `The Messenger / 急速羽翼`
2. Header Lockup：`Symbol + 文文。速报 + 幻想乡活动情报总览`
3. Editorial Stamp：用于 `今日 / 精选 / 已更新` 等轻量品牌标记

Logo 当前问题不在符号本身，而在缺少系统级使用方式。

## 10. 视觉与内容策略

### 10.1 保留项

- theme-newspaper 单主题
- 米色纸张背景
- 粗黑边框
- 报纸式栏目语言
- 红色印章式强调元素

### 10.2 调整项

- 降低装饰元素的泛滥程度
- 提升正文与元信息的可读性和层级
- 让首页更像有秩序的“头版”，而不是被一组重卡片堆出来的版面

### 10.3 内容语气

建议内容语气统一为：

- 编辑型
- 简洁
- 有判断力
- 不做平台式吆喝

## 11. 响应式策略

### 11.1 桌面端

- 保持明确的头版感与左右分栏结构
- Hero 采用双核心布局
- Quick Rail 作为辅助信息列存在

### 11.2 移动端

- 采用任务优先顺序
- 品牌与搜索必须前置
- 地区筛选不应在首屏主导版面
- 时间轴压缩为预览型卡片流

## 12. 可访问性与质量要求

### 12.1 可访问性

必须在重构中补齐：

- Header / Footer 中可见文本与 accessible name 一致
- 所有交互控件保留清晰 focus-visible
- 对比度满足可读要求
- 保留 skip link
- 保留 aria-live 的异步反馈区域

### 12.2 质量要求

- Hero 首屏必须在不滚动情况下说明产品价值
- 首页工具入口必须在首屏或首屏后第一层出现
- 所有 CTA 文案必须具体，不使用模糊按钮文案
- 时间轴在首页中必须是预览，而非完整主体

## 13. 埋点建议

建议埋点以下行为：

- Hero 主 CTA 点击
- Hero 次 CTA 点击
- Header 搜索使用
- Command Bar 筛选使用
- 首页到活动详情点击
- 首页到 `/events` 点击
- 首页到 `/circles` 点击
- 首页到 `/feedback` 点击
- 角色入口点击

## 15. 最终结论

本次 landing 重构的核心，不是简单优化某一个 Hero 或换一套样式，而是重新定义 AyaFeed 首页的职责。

重构后，首页应当做到：

- 首屏先讲清 AyaFeed 是什么
- 第二层立刻给出搜索与筛选入口
- 中段用编辑方式展示重点档期与更新
- 下段保留时间轴作为差异化内容资产预览
- 底部提供观众、社团、主办方的明确参与路径
- Header、Footer、Logo 一起形成统一品牌系统

如果本 PRD 获得确认，下一阶段应进入实现规划，输出组件级实施计划与分步改造方案。
